Overview

Unit testing is a software testing method where individual units or components of software are tested. The purpose is to validate that each unit of the software performs as designed.

Unit Testing Theory

Introduction to Unit Testing

Unit Testing is the process of testing individual components or units of software to verify their correctness.

Why Unit Testing is Important:

  • Early detection of bugs and issues
  • Improves code quality and design
  • Serves as documentation
  • Facilitates refactoring

Core Concepts:

  • Test isolation and independence
  • Test fixtures and setup
  • Mock objects and stubs
  • Assertions and verifications

Testing Pyramid

UI Tests
Integration Tests
Unit Tests

Layer Explanation:

  • UI Tests: End-to-end testing of the application
  • Integration Tests: Testing multiple components together
  • Unit Tests: Testing individual components in isolation

Key Principles

Test Independence

Each test should be able to run independently of others

Single Assert

One assertion per test method for clarity and focus

Repeatability

Tests should produce the same results every time they run

Simplicity

Tests should be simple and clear to understand

Fast Execution

Tests should run quickly to enable frequent execution

Maintenance

Tests should be maintained with the same care as production code

Advantages and Disadvantages

Benefits Challenges
  • Early bug detection
  • Improved code quality
  • Better design practices
  • Documentation through tests
  • Confidence in refactoring
  • Initial time investment
  • Learning curve
  • Maintenance overhead
  • False sense of security
  • Test suite complexity

Testing Best Practices

  • Arrange: Set up the test conditions
  • Act: Execute the code being tested
  • Assert: Verify the results

  • Use descriptive test names
  • Follow pattern: UnitOfWork_StateUnderTest_ExpectedBehavior
  • Be specific about what's being tested

  • Group related tests together
  • Follow consistent folder structure
  • Separate test utilities and helpers

  • Aim for meaningful coverage, not just numbers
  • Focus on critical business logic
  • Consider both positive and negative test cases

Best Practices

  • Follow AAA pattern (Arrange-Act-Assert)
  • Keep tests focused and concise
  • Use meaningful test names
  • Maintain test code quality

Testing the Bootstrap Page Locally

Prerequisites

  • Python 3.x installed on your system
  • A modern web browser (Chrome, Firefox, Safari)
  • Basic familiarity with terminal/command prompt

Starting the Local Server

Open your terminal and navigate to the project directory. Then run:

python3 -m http.server 8000
The server will start and listen on port 8000

Accessing the Page

Open your web browser and navigate to:

  • http://localhost:8000
  • http://127.0.0.1:8000

Testing Checklist

  • ✓ Click through all navigation tabs
  • ✓ Test carousel auto-rotation and manual navigation
  • ✓ Verify responsive design (resize browser)
  • ✓ Check code snippet highlighting
  • ✓ Test mobile view (F12 for device emulation)
  • ✓ Verify Bootstrap icons display

Troubleshooting

If the server won't start:

  • Check if port 8000 is already in use
  • Try a different port number
  • Ensure you have necessary permissions

If the page doesn't load:

  • Verify the server is running
  • Check your browser's console for errors
  • Clear browser cache and refresh

Common Assertions


                // Basic Assertions
                assertEquals(expected, actual);
                assertTrue(condition);
                assertFalse(condition);
                assertNull(object);
                assertNotNull(object);

                // Advanced Assertions
                assertThrows(Exception.class, () -> {
                    // code that should throw exception
                });

                assertAll("group name",
                    () -> assertEquals(2, 1 + 1),
                    () -> assertTrue(4 > 3)
                );

                assertTimeout(Duration.ofSeconds(1), () -> {
                    // code that should complete within 1 second
                });
Pro Tips:
  • Always include descriptive messages in assertions
  • Use assertAll for multiple related assertions
  • Consider using assertTimeout for performance-critical tests

Parameterized Tests

Run the same test with different parameters:


                @ParameterizedTest
                @ValueSource(strings = {"racecar", "radar", "able was I saw elba"})
                void palindromes(String candidate) {
                    assertTrue(isPalindrome(candidate));
                }

                @ParameterizedTest
                @CsvSource({
                    "apple,  5",
                    "banana, 6",
                    "orange, 6"
                })
                void testLength(String word, int length) {
                    assertEquals(length, word.length());
                }

                @ParameterizedTest
                @EnumSource(TimeUnit.class)
                void testTimeUnitMinutes(TimeUnit unit) {
                    assertTrue(unit.toString().contains("MINUTES"));
                }

Advanced JUnit 5 Features

1. Nested Tests

                @DisplayName("Calculator Tests")
                class CalculatorTests {
                    private Calculator calculator;

                    @Nested
                    @DisplayName("Addition Operations")
                    class AdditionTests {
                        @Test
                        @DisplayName("Positive Numbers")
                        void testPositiveAddition() {
                            assertEquals(5, calculator.add(2, 3));
                        }

                        @Test
                        @DisplayName("Negative Numbers")
                        void testNegativeAddition() {
                            assertEquals(-5, calculator.add(-2, -3));
                        }
                    }

                    @Nested
                    @DisplayName("Division Operations")
                    class DivisionTests {
                        @Test
                        @DisplayName("Division by Zero")
                        void testDivisionByZero() {
                            assertThrows(ArithmeticException.class, 
                                () -> calculator.divide(1, 0));
                        }
                    }
                }
2. Dynamic Tests

                @TestFactory
                Collection dynamicTests() {
                    return Arrays.asList(
                        dynamicTest("Add Test", 
                            () -> assertEquals(4, calculator.add(2, 2))),
                        dynamicTest("Multiply Test", 
                            () -> assertEquals(6, calculator.multiply(2, 3)))
                    );
                }

Test Suites

Group multiple test classes:


                @Suite
                @SelectPackages("com.example.tests")
                @IncludeClassNamePatterns(".*Test")
                public class AllTests {
                    // Suite will include all classes that end with "Test"
                    // in the com.example.tests package
                }

                // Or specify classes explicitly
                @Suite
                @SelectClasses({
                    UserTest.class,
                    OrderTest.class,
                    PaymentTest.class
                })
                public class CommerceTests {
                }
Suite Features:
  • Group related tests together
  • Configure suite-wide settings
  • Control test execution order
  • Filter tests using tags

Python Local Server Setup

Features

  • ✓ Customizable port configuration (default: 8000)
  • ✓ Port availability checking
  • ✓ Python version compatibility check
  • ✓ Colorized terminal output
  • ✓ Helpful error messages and instructions

Server Script


                        #!/bin/bash

                        # Colors for terminal output
                        RED='\033[0;31m'
                        GREEN='\033[0;32m'
                        YELLOW='\033[1;33m'
                        NC='\033[0m' # No Color

                        # Default port
                        PORT=8000

                        # Process command line arguments
                        while getopts "p:h" opt; do
                            case ${opt} in
                                p )
                                    PORT=$OPTARG
                                    if ! [[ "$PORT" =~ ^[0-9]+$ ]] || [ "$PORT" -lt 1024 ] || [ "$PORT" -gt 65535 ]; then
                                        echo -e "${RED}Error: Port must be a number between 1024 and 65535${NC}"
                                        exit 1
                                    fi
                                    ;;
                                h )
                                    show_help
                                    exit 0
                                    ;;
                            esac
                        done

                        # Check Python and start server
                        if command -v python3 &> /dev/null; then
                            PYTHON_CMD="python3"
                        else
                            PYTHON_CMD="python"
                        fi

                        $PYTHON_CMD -m http.server $PORT

Usage Instructions

Basic Usage:

./cmdToRun-Python-Server-Local.sh

Custom Port:

./cmdToRun-Python-Server-Local.sh -p 8080

Show Help:

./cmdToRun-Python-Server-Local.sh -h
After starting the server, access your web pages at:
  • http://localhost:8000
  • http://127.0.0.1:8000

Troubleshooting

Common Issues:

  • Port already in use: Try a different port using -p option
  • Permission denied: Make sure the script is executable (chmod +x)
  • Python not found: Ensure Python is installed on your system
