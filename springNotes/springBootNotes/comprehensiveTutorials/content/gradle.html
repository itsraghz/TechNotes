Gradle Build System

A powerful and flexible build automation tool for modern software development

Gradle is a powerful and flexible build automation tool that helps developers manage project dependencies, compile code, run tests, and package applications. It combines the best features of Apache Ant and Maven while introducing a Groovy-based Domain Specific Language (DSL) for declaring builds.

  • High Performance: Incremental builds and build cache capabilities significantly reduce build times
  • Flexibility: Highly customizable with support for multiple languages and platforms
  • Developer Experience: Clear and concise build scripts using Groovy or Kotlin DSL
  • Extensibility: Rich plugin ecosystem and easy integration with existing tools

  • Incremental builds
  • Build cache support
  • Parallel task execution
  • Dependency management
  • Multi-project builds
  • Custom task types

  • gradle build Compiles and packages your project, running all tests
  • gradle clean Deletes the build directory and all built files
  • gradle test Runs all tests in your project
  • gradle tasks Lists all available tasks in the current project

Gradle provides different dependency configurations to control how dependencies are used in your project.

Common Dependency Configurations

  • implementation

    Dependencies required for compilation and runtime. Not exposed to dependent projects.

    implementation 'org.springframework.boot:spring-boot-starter-web:2.7.0'
  • compileOnly

    Dependencies needed only at compile time, not included in runtime classpath.

    compileOnly 'org.projectlombok:lombok:1.18.24'
  • testImplementation

    Dependencies used for testing only, not included in the main artifact.

    testImplementation 'junit:junit:4.13.2'
  • runtimeOnly

    Dependencies required only at runtime, not needed for compilation.

    runtimeOnly 'org.postgresql:postgresql:42.3.1'
